成員介紹

張瑋鐘
分工比例: 31%
這學期的多媒體程式設計課程對我而言，是一段充滿挑戰但也極具成就感的旅程。身為負責 JavaScript 邏輯與檔案整合的角色，我從中深刻體會到前端開發不僅是畫面的呈現，更是資料流動的控管。起初在實作購物車與收藏功能時，我面臨了在缺乏後端資料庫支援下，網頁重新整理或跳轉頁面後資料就會重置的難題。這讓我必須跳脫原本單純操作 DOM 的思維。我成功模擬了資料庫的持久化功能，解決了跨頁面資料同步與導覽列即時更新的問題。這個過程讓我從過去單純依賴教學範例，轉變為能主動思考前端資料生命週期的架構設計。雖然 JavaScript 的邏輯除錯仍具難度，但看著自己寫出的程式碼能讓網站功能流暢運作，這份實作經驗不僅提升了我的技術能力，更成為我未來深入探索前端工程的重要基石。
陳柏樺
分工比例: 32%
這學期的多媒體程式設計課對我來說是一門充滿挑戰卻收穫很多的課程，雖然在實作與小考中常會卡關，整體仍有進步空間，但卻是我所有輔系課程中最喜歡的一門。透過一步步使用 HTML、CSS 與 JavaScript 架設網頁，我真正理解了前端是如何被建構出來的，也從過去依賴教學與 AI，轉變為能從空白檔案開始思考結構並實際寫出程式碼。雖然 JavaScript 對我來說仍較困難，需要更多時間練習，但這堂課讓我不再只是看得懂程式，而是開始真正寫得出來，這份成長也成為我未來繼續學習前端的重要動力。 在製作電腦版電商網站的過程中，我發現即使只針對桌機畫面設計，CSS 版面仍然容易出現跑掉的情況。當畫面解析度不同、瀏覽器視窗大小改變，或是實際商品資料與預期不一致時，原本看起來正常的版面就可能出現對齊錯誤、內容溢出或區塊間距不一致的問題。
黃恩國
分工比例: 25%
在製作期末專題的過程中，我深刻感受到同學們都好厲害。他們都能夠自學並運用課程中沒有教過的內容，讓整體報告在完成基本功能之外，品質也提升了不少。相較之下，我也意識到自己在前端能力上仍有很大的進步空間。由於我是輔系生，這次在專題的程式撰寫部分幾乎幫不上忙，但我希望自己可以盡力在其他方面做出貢獻，例如協助檢視內容是否有需要改進的地方、整理文字資料，以及完成自己能力所及的工作。同時，我也嘗試理解組員們所負責的程式內容，從中學習。整體而言，我很慶幸能夠和一群認真又給力的組員一起合作，讓這次的專題能愉快的完成。
簡揚軒
分工比例: 12%
這學期的多媒體程式設計課程，對我而言是一場從「視覺思維」轉向「邏輯架構」的訓練。在參與製作電商網站的過程中，我深刻體會到網頁設計並不像平面設計那樣，將元素擺放好就會永遠固定不動。相反地，程式碼構建出的是一個「動態的容器」，必須要能包容各種變數。 以前瀏覽網頁時，覺得一切理所當然，自己動手寫才發現，要讓一個版面在不同情況下都保持美觀，需要極為細膩的 CSS 佈局邏輯。這門課最大的收穫，不僅是學會了語法，更是培養了「預判問題」的能力，在寫下一行程式碼前，先思考這個區塊如果文字變多怎麼辦？如果螢幕變窄怎麼辦？這種在挫折中不斷除錯、修正的過程，雖然痛苦，但當看到網站最終能良好呈現時，那份成就感是其他課程難以比擬的。 在開發網站的過程中，我遇到了一些技術挑戰，這也讓我對 CSS 的運作機制有了更深的理解： 1. 真實資料導致的版面崩壞 在初期，我使用固定的測試文字與圖片，畫面看起來非常整齊。但當我們嘗試填入真實的商品資料時，問題就浮現了： 卡片高度不一： 由於每項商品的名稱長短差異，導致並排的商品卡片高度參差不齊，下方的按鈕也無法對齊，那就破壞了原本的視覺。 圖片比例變形： 即使設定了圖片寬度，但因為原始圖片的長寬比不一致，硬性縮放後常常造成圖片變形失真。 2.解析度適應不良： 雖然是做電腦版，但在我的螢幕上調整好的間距，換到解析度較小的筆電上時，版面就會變得過於擁擠甚至重疊。這讓我學到在佈局時，不能過度依賴絕對的 px 數值，而需要更多考慮相對單位與整體結構的彈性。
